# Awesome Chocolates SQL Project

## 1. Project Overview

The Awesome Chocolates SQL Project is a structured database design and data analysis project developed using SQL. It models a real-world business scenario involving chocolate sales across multiple regions, products, and sales teams.

The project focuses on building a relational database from scratch, defining appropriate schemas, and preparing the dataset for analytical queries. It serves as a foundational project for understanding how SQL can be applied to solve business problems through structured data.

---

## 2. Business Objective

The primary objectives of this project are:

* To analyze sales performance across different geographical regions
* To evaluate the contribution of individual salespersons and teams
* To understand product-level performance and category trends
* To create a scalable dataset that can be used for business intelligence and reporting

---

## 3. Database Schema

The database is composed of four core tables designed with relational integrity in mind.

### 3.1 GEO (Geographical Data)

This table stores information related to countries and their respective regions.

| Column Name | Description                                          |
| ----------- | ---------------------------------------------------- |
| GEOID       | Unique identifier for each geographical location     |
| GEO         | Name of the country                                  |
| REGION      | Region classification (e.g., APAC, Americas, Europe) |

---

### 3.2 PEOPLE (Sales Team)

This table contains details about the sales workforce.

| Column Name | Description                                 |
| ----------- | ------------------------------------------- |
| SPID        | Unique salesperson identifier (Primary Key) |
| SALESPERSON | Name of the salesperson                     |
| TEAM        | Assigned team name                          |
| LOCATION    | City or operational location                |

---

### 3.3 PRODUCTS (Product Details)

This table stores all product-related information.

| Column Name  | Description                                 |
| ------------ | ------------------------------------------- |
| PID          | Unique product identifier (Primary Key)     |
| PRODUCT      | Name of the product                         |
| CATEGORY     | Product category (e.g., Bars, Bites, Other) |
| SIZE         | Product size classification                 |
| COST_PER_BOX | Cost associated with each box               |

---

### 3.4 SALES (Transaction Data)

This is the central fact table that captures all sales transactions.

| Column Name | Description                           |
| ----------- | ------------------------------------- |
| SPID        | Salesperson identifier (Foreign Key)  |
| GEOID       | Geographical identifier (Foreign Key) |
| PID         | Product identifier (Foreign Key)      |
| SALEDATE    | Date of transaction                   |
| AMOUNT      | Total revenue generated               |
| CUSTOMERS   | Number of customers involved          |
| BOXES       | Total number of boxes sold            |

---

## 4. Relationships Between Tables

* The SALES table acts as the central transactional table.
* It is connected to:

  * PEOPLE via SPID
  * GEO via GEOID
  * PRODUCTS via PID

This relational structure allows efficient querying using JOIN operations to generate meaningful business insights.

---

## 5. Key Features

* Creation of a structured relational database using SQL
* Implementation of primary and foreign key relationships
* Organized schema design for scalability and readability
* Realistic dataset suitable for analytical queries
